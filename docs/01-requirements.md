# 需求分析

## 核心功能

- **用户系统**：注册/登录/登出，密码 bcrypt 加盐哈希存储
- **留言系统**：发布顶级留言、回复任意层级留言（无限嵌套）、删除留言（软删除 + 子回复保留）
- **数据字段**：id、parentId、userId、userName（冗余，避免 N+1）、content、isDeleted、createdAt
- **排序展示**：顶级留言按时间倒序，子回复按时间正序，最大缩进 4 层

## 非功能需求

| 类别 | 要求 |
|------|------|
| XSS 防护 | 前后端双重转义，React JSX 转义 + 后端 escapeHtml |
| 密码安全 | bcrypt 加盐哈希 |
| 越权控制 | 删除/编辑时校验 req.userId === comment.userId |
| SQL 注入 | Prisma ORM 参数化查询 |
| 性能 | userName 冗余存储避免 N+1；parentId/createdAt 索引 |
| 工程规范 | .env 管理敏感配置；核心算法单元测试；Docker 一键部署 |

## 边界条件

- 父留言删除后子回复保留，显示"该留言已被删除"
- 未登录用户可浏览但不可操作
- 深度嵌套超 4 层停止物理缩进
- 空内容前后端双重校验

## 验收标准

- 未登录可浏览 / 注册登录登出 / 发表回复删除 / 越权拦截 / 软删除保留子回复 / 无限层级正确渲染 / XSS 不可执行
